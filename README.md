# Сайт завода — фундамент проекта

Семейский судостроительно-судоремонтный завод. Статический сайт (HTML/CSS/JS без сборщика) — открывается прямо в браузере, легко доработать и разместить на любом хостинге.

## Структура

```
index.html        — разметка всех разделов
css/styles.css     — все стили (токены, компоненты, адаптив)
js/main.js         — интерактив (меню, ripple, скролл-анимации, форма)
```

## Что уже сделано

- **Разделы**: О нас, Услуги, Проекты, Контакты, Связь с нами (+ герой-блок и футер), все с якорями в навигации.
- **Дизайн**: цвета воды/стали/ржавчины (см. переменные `--color-*` в начале `styles.css`), шрифты Oswald (заголовки) + PT Sans (текст, оптимизирован под кириллицу, не режет глаза).
- **Фирменный элемент**: круглая «заводская табличка» в духе судостроительных бирок — 1908 / 118 лет на воде.
- **Анимация волн**: волна — это повторяющийся (repeat-x) SVG-тайл шириной 720px, который бесконечно едет через `background-position`. Именно поэтому она зациклена без единого шва и не «дёргается» — каждый цикл идентичен предыдущему. Цвет и скорость настраиваются в блоке `6. ВОЛНЫ-РАЗДЕЛИТЕЛИ` в `styles.css`.
- **Проекты**: раздел `#projects` — сетка карточек с заглушкой под фото, меткой (тип работ · год), названием и описанием. Три карточки-примера, дублируйте `<article class="project-card">` под каждый новый проект.
- **Интерактив и анимации**: единый «живой» слой оформления — прогресс-бар прокрутки вверху, подсветка активного пункта меню при скролле (scrollspy), каскадное появление блоков и карточек, счётчики-цифры в разделе «О нас», линии-акценты под заголовками, отрисовка кольца заводской таблички, ховер-эффекты карточек и кнопок (блик, подъём, акцентная полоса), индикатор «листайте вниз», лёгкий параллакс героя, ripple-эффект на кнопках, мобильное меню, форма с базовой валидацией. Всё уважает системную настройку «уменьшить движение» (`prefers-reduced-motion`).
- **Адаптив**: брейкпоинты на 980px / 760px / 480px, фокус-состояния для клавиатуры, уважение `prefers-reduced-motion`.

## SEO и шаринг (сделано)

- **Мета-теги**: title, description, canonical, theme-color, Open Graph и Twitter-карточка — ссылка на сайт разворачивается превью в мессенджерах и соцсетях.
- **Иконки**: `favicon.svg` (векторная, с фирменным якорем) + `site.webmanifest` (имя, цвета, установка как PWA).
- **Обложка для превью**: `assets/og-cover.svg` (1200×630). ⚠ Часть платформ (Telegram и др.) **не показывают SVG** в превью — для максимальной совместимости экспортируйте её в `assets/og-cover.jpg` и поменяйте пути `og:image`/`twitter:image` в `<head>`.
- **Индексация**: `robots.txt` + `sitemap.xml`.
- **Структурированные данные**: JSON-LD `LocalBusiness` в `<head>` (адрес, телефон, часы работы, год основания) — помогает поисковикам показать карточку организации.

> ⚠ **Домен-заглушка `https://sssz.kz`** прописан в `<head>` (canonical/og/JSON-LD), `robots.txt` и `sitemap.xml`. Замените его на реальный адрес сайта во всех трёх местах.

## Фоновое видео в герой-блоке

В первый экран (`.hero`) встроено фоновое видео. Чтобы оно появилось — положите файл в папку `assets/`:

- `assets/hero.mp4` — основной файл (H.264/MP4, поддерживается везде);
- `assets/hero.webm` — по желанию, легче по весу для современных браузеров;
- `assets/hero-poster.jpg` — постер (первый кадр), показывается, пока видео грузится и на мобильных с отключённым автозапуском.

Рекомендации: длительность 10–20 сек, зациклённый ролик без резких склеек, вес **до ~5–8 МБ** (иначе первый экран будет долго грузиться), разрешение 1920×1080. Видео идёт без звука (`muted`) и само зацикливается.

Всё уже настроено: автозапуск, зацикливание, тёмный оверлей для читаемости текста, десатурация под палитру сайта и режим «меньше движения» (`prefers-reduced-motion` — видео прячется, остаётся постер/градиент). **Пока файла нет — герой корректно показывает прежний градиент**, ничего не сломается.

## Что нужно заменить перед публикацией

1. **Контакты** (`index.html`, секция `#contacts`) — сейчас адрес, телефон и email — заглушки. Замените на реальные (не забудьте синхронно поправить телефон/email в блоке JSON-LD в `<head>`).
2. **Карта** — вставлен рабочий iframe Google Карт по адресу из секции контактов. ⚠ Проверьте, что пин встал верно; при смене адреса замените параметр `q=` в `src` (адрес в URL закодирован — можно просто вписать новый через любой URL-энкодер или сервис «Google Карты → Поделиться → Встроить карту»).
3. **Форма** (`js/main.js`, блок 6) — сейчас форма ничего никуда не отправляет, только показывает сообщение. Нужно подключить backend (например, отправку на email через свой сервер, или сервис вроде Formspree/Getform).
4. **Тексты** — часть копирайтинга (описание услуг, история завода) написана как пример по заданным фактам (1908 год, 118 лет). Дополните реальными формулировками и фактами по мере готовности контента.
5. **Проекты** (`index.html`, секция `#projects`) — три карточки-заглушки. Для каждого проекта:
   - замените текст в `<span class="project-meta">`, `<h3>` и `<p>` внутри `.project-body`;
   - вместо блока-заглушки `<div class="project-photo">...</div>` вставьте реальное фото, например:
     ```html
     <div class="project-photo">
       <img src="images/project-1.jpg" alt="Название судна" style="width:100%;height:100%;object-fit:cover;">
     </div>
     ```
     (создайте папку `images/` рядом с `index.html` и положите туда фотографии).
   - чтобы добавить ещё один проект — скопируйте весь блок `<article class="project-card">...</article>` и вставьте его в `.projects-grid`.

## Как расширять

- Новый раздел = новая `<section class="section section-light|section-dark">` в `index.html` + пункт в `.main-nav` и `.footer-nav`.
- Новая карточка услуги = скопировать `.service-card` внутри `.cards-grid`.
- Цвета/шрифты меняются в одном месте — блок `:root` в начале `styles.css`.

Дальше можно добавлять: галерею объектов/судов, страницу с новостями, отдельные страницы услуг, интеграцию с картой и формой.
